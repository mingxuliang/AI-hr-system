# 安全说明

## 部署清单

- 设置强随机 `SECRET_KEY`
- 修改 `INITIAL_ADMIN_PASSWORD`，首次登录后轮换管理员密码
- PostgreSQL 使用独立账号和强密码
- `CORS_ORIGINS` 仅允许可信域名
- 使用 HTTPS 与反向代理
- 生产环境保持 `ENABLE_API_DOCS=false`
- 对上传文件做容量与类型限制
- 对简历、录音、评价、Offer 等敏感数据制定保留策略
- 确认 LLM 服务商数据处理政策符合组织要求

## 报告问题

请通过内部渠道联系维护人员，勿在公开场合披露可利用漏洞。说明影响范围、复现步骤与相关日志即可。
