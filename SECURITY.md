# Security Policy

## 支持范围

当前主分支会接收安全修复。正式发布版本后，建议以 GitHub Release 标记受支持版本。

## 报告安全问题

请不要在公开 Issue 中披露可利用漏洞。可以通过仓库维护者公开的安全联系方式报告，并尽量包含：

- 影响范围
- 复现步骤
- 相关日志或请求样例
- 可能的修复建议

## 部署安全清单

- 设置强随机 `SECRET_KEY`。
- 修改 `INITIAL_ADMIN_PASSWORD`，首次登录后及时轮换管理员密码。
- 为 PostgreSQL 使用独立账号和强密码。
- 仅允许可信域名出现在 `CORS_ORIGINS`。
- 使用 HTTPS 和反向代理访问控制。
- 对上传文件进行容量、类型和恶意内容限制。
- 对简历、录音、评价和 Offer 等敏感数据设置保留策略。
- 确认 LLM 服务商的数据处理政策符合组织要求。
